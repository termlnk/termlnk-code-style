# 02. 依赖注入 (DI) 规范

## 接口与标识符:同名常量模式

接口名 = DI 标识符常量名,标识符值用 `'<包名>.<服务名>.(service|controller)'`:

```typescript
// ✅ 正确:接口与标识符同名
export interface ITerminalService {
  readonly sessions$: Observable<ITerminalSession[]>;
  createSession(config: ISessionConfig): ITerminalSession;
  closeSession(sessionId: string): void;
}

export const ITerminalService = createIdentifier<ITerminalService>('terminal.terminal.service');
```

更多示例:

```typescript
export const ICommandService = createIdentifier<ICommandService>('core.command.service');
export const IThemeService = createIdentifier<IThemeService>('core.theme.service');
export const IHostService = createIdentifier<IHostService>('terminal.host.service');
```

## 构造函数注入

**全项目通用规范**：所有 class（Service / Controller / Plugin / Repository）的 DI 注入字段**必须**直接写在构造参数上作为 TypeScript parameter property（`@Decorator() <可见性> readonly _x: T`），**禁止**先在类体内单独声明字段、再在构造函数体内 `this._x = x` 显式赋值的拆分写法。后者是反模式 —— 它增加无意义的胶水代码并破坏模板一致性。

两套可见性约定（差异仅在 Plugin 与其他类型间）：

- **Service / Controller / Repository**：所有注入参数 `private readonly` + `_` 前缀
- **Plugin**：`_injector` 用 `protected readonly`（生命周期钩子 `onStarting/onRendered/onSteady` 需访问 child injector，与基类 `Plugin` 成员可见性对齐）；`_configService` 用 `private readonly`（仅构造期 `setConfig` 用一次）。业务字段如 `_config` 也是 `private readonly`

```typescript
// ✅ Service / Controller / Repository:所有注入参数 private readonly + _ 前缀
export class TerminalSessionService extends Disposable implements ITerminalSessionService {
  constructor(
    @ICommandService private readonly _commandService: ICommandService,
    @ILogService private readonly _logService: ILogService,
    @Inject(Injector) private readonly _injector: Injector,
    @Inject(LifecycleService) private readonly _lifecycleService: LifecycleService,
  ) {
    super();
  }
}

// ✅ Plugin:_injector 用 protected readonly,_configService 用 private readonly
@DependentOn(UIPlugin, RPCClientPlugin)
export class TerminalUIPlugin extends Plugin {
  static override pluginName = TERMINAL_UI_PLUGIN_NAME;

  constructor(
    private readonly _config: Partial<ITerminalUIConfig> = defaultPluginConfig,
    @Inject(Injector) protected readonly _injector: Injector,
    @IConfigService private readonly _configService: IConfigService,
  ) {
    super();
  }
}
```

### 反模式 — 拆字段 + 体内赋值

**全平台、全包类型一律禁止**。即使下面"Mobile babel 工具链债"段落里描述的临时拆开写法，也是债务而非规范本身的变体。

```typescript
// ❌ 反模式：把 DI 注入字段拆成类体内单独声明 + 构造函数体内显式赋值
export class XxxService extends Disposable {
  private readonly _logService: ILogService;
  private readonly _commandService: ICommandService;

  constructor(
    @Inject(ILogService) logService: ILogService,
    @ICommandService commandService: ICommandService
  ) {
    super();
    this._logService = logService;
    this._commandService = commandService;
  }
}
```

### 注入规则

- 接口标识符直接用作装饰器：`@ICommandService`
- 具体类使用 `@Inject()`：`@Inject(Injector)`、`@Inject(LocaleService)`
- **可选依赖**使用 `@Optional()`（见下方专章）
- Service / Controller / Repository 注入参数：`private readonly` + `_` 前缀
- Plugin 类的 `_injector`：`protected readonly` + `_` 前缀；`_configService` 与业务字段（如 `_config`）：`private readonly` + `_` 前缀

### Mobile babel 工具链债 — 不是规范的例外

下述描述是工具链 bug 的临时绕行方案，**不**改变上面的通用规范本身。`apps/mobile/babel.config.js` 的 `babel-plugin-parameter-decorator` 与 `@babel/plugin-transform-typescript` 串接顺序有已知 bug：mobile 端的 **service / repository / controller** 类如果按规范写（参数装饰器 + parameter property），装饰器元数据会在 TS 改写参数属性前被吞掉，运行时注入字段全部 `undefined`。

| 范围 | 当前必须怎么写 | 是否符合规范 |
|------|---------------|------------|
| 桌面 / web / packages/* 全部类型 | 参数属性形式 | ✅ |
| Mobile **plugin** | 参数属性形式（实测 babel 链对 plugin 不影响） | ✅ |
| Mobile **service / repository / controller** | 拆字段 + 体内赋值 | ❌ 反模式，但目前**必须**这样写避免运行时 bug |

Mobile service / repo / controller 的拆字段写法属于 babel 链上的技术债，待工具链修复后回归规范。详见 memory [[mobile-babel-no-param-property-decorator]]。

## @Optional 装饰器 — 可选依赖

某依赖可能未被注册时,用 `@Optional()`,未找到时参数为 `undefined`,不会抛异常:

```typescript
import { Optional } from '@termlnk/core';

export class SyncController extends Disposable {
  constructor(
    @ICommandService private readonly _commandService: ICommandService,
    @Optional(DataSyncPrimaryController)
    private readonly _dataSyncPrimaryController?: DataSyncPrimaryController,
  ) {
    super();
    this._dataSyncPrimaryController?.registerSyncingMutations(SaveMutation);
  }
}
```

`@Inject` / `@Optional` / 接口标识符 (`@IXxxService`) 三者不可混用。

## 服务注册四种形态

```typescript
// 接口 → 实现
[ITerminalService, { useClass: TerminalService }]

// 直接注册类(controller 等需立即实例化的)
[TerminalController]

// 工厂模式
[IServiceId, { useFactory: (accessor: IAccessor) => new ServiceImpl(accessor.get(IDep)) }]

// 值注册
[IToken, { useValue: someValue }]
```

## 反模式

```typescript
// ❌ 全局 singleton 模式
export const themeService = new ThemeService();

// ❌ 通过参数传递 service
function doWork(themeService: IThemeService) { ... }

// ❌ 接口标识符与接口不同名
export const THEME_SERVICE = createIdentifier<IThemeService>('...');
```
